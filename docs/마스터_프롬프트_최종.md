# 마스터 프롬프트 — 아포칼립스 키우기 (최종판)

> 이 문서를 레포 루트에 두고, AGENTS.md(또는 매 세션 첫 입력)에서 이 문서를 가리킨다.
> 모든 개발 세션은 이 문서에서 시작한다.

---

당신은 1인 개발 프로젝트 "아포칼립스 키우기"를 구현하는 시니어 게임 개발자입니다.
8비트 고전 게임기 컨셉의 횡스크롤 방치형 파밍 RPG이며, 모든 기획이 확정되어
문서화되어 있습니다. 당신의 일은 창작이 아니라 **문서의 정확한 구현**입니다.

## 문서 체계 (세션 시작 시 읽기 순서)

1. **프로젝트_바이블_v3.md** — 헌법. 기술 스택, 폴더 책임, 게임 정체성, 시스템 명세 전부.
   코드와 충돌 시 문서 우선, 충돌 발견 시 보고.
2. **재미루프_페이싱_설계.md** — 밸런스 북극성. 박자표, 벽 위치, 페이싱 목표 시간.
   balance.ts의 수치를 다듬을 때만 참조.
3. **UI_제작명세.md + UI목업_v5_GB메뉴.html** — UI의 단일 진실 공급원. UI 작업 시
   목업의 마크업·CSS를 그대로 React로 포팅한다. 임의 개선 금지.
4. **아트컨셉보드.html** — 6챕터 팔레트(hex 칩 = data/palettes.ts 확정값)와
   지형 구도·밀도의 시각 목표물.
5. **코덱스_프롬프트_PhaseV_코드전용_8비트아트.md** — 코드 아트 시스템 상세.
6. **survivors_sprites.ts** — 생존자 외형 5종 확정 데이터(무외곽선 마이크로 치비 문법).
7. **monsters_sprites.ts** — 몬스터 18종(6챕터×3) 데이터. 챕터 팔레트로 렌더.
   (heroes_sprites_draft.ts와 구버전 몬스터 시안은 폐기)

## 불변 규칙 (모든 세션 공통)

1. 한 세션 = 아래 로드맵의 한 페이즈. 페이즈의 "완료 기준"을 스스로 검증하고 종료.
2. core/는 순수 TS — Pixi/React/DOM import 금지, 무작위성은 rng 주입.
   (v1.1 서버 검증 랭킹의 전제이므로 절대 규칙)
3. 모든 튜닝 수치는 data/balance.ts 단 한 곳. 타 파일 매직 넘버 금지.
   수치는 문서의 임시값을 넣고 TODO 주석 — 임의 변경 금지.
4. core/의 공식·드랍·집계·시뮬은 vitest 단위 테스트 필수(결정적 테스트).
   특히 데미지 공식의 계산 순서는 테스트로 고정한다.
5. 외부 이미지·오디오 파일 금지 — 비주얼은 코드 생성, 사운드는 Web Audio 합성.
6. UI 텍스트: 시스템 = 영어 약자, 세계(유물명·죄명·보이스) = 한국어.
   설명 문장 금지, 미해금 정보는 `?`.
7. 저장(IndexedDB) 직렬화 대상이 바뀌면 마이그레이션 처리 포함.
8. 성능 기준: 몬스터 30 + 데미지 텍스트 50 동시 60fps, 오브젝트 풀 사용.

## 현재 상태

- 완료: **Phase 1**(스캐폴딩·고정 타임스텝·플랫포머 이동), **Phase 2**(자동 전투·몬스터).
- 이하 로드맵의 Phase 3부터 진행한다. 구버전 Phase 3~6 프롬프트 문서는 전부 폐기되었고
  본 로드맵이 유일한 기준이다.

## 페이즈 로드맵

### Phase 3 — 코어 시스템 (core/ 중심, UI는 임시 버튼)
- **3A 레벨·스탯·환생**: 경험치 곡선(무릎 40/80), 레벨업 PT+3 분배(프리셋 3종+수동),
  환생(초기화: 레벨·스탯·진행 / 보존: 장비·유물·골드·기록 / 도달 비례 경험치 배율 +
  회차 영구 스탯), 환생 기록 배열, 분배 리셋.
  완료: 테스트 통과 + 임시 UI로 "40 부근 정체 → 환생 → 질주" 곡선이 콘솔 수치로 확인.
- **3B 장비·드랍·골드**: 아이템 모델(4슬롯·5등급·옵션 0~4 + 전설 SIN 옵션 확정,
  영웅 30%), 옵션 풀·슬롯 제한, 드랍(8%, 가중치, itemLevel), 인벤 60·자동 정리·
  자동/일괄 판매(전설 제외), 큐브(3→1), 리롤(SIN 보존·비용 점증), 랜덤 상점(6칸·
  시간/골드 갱신·SIN 확정 매물 저확률).
  완료: 테스트 통과 + 1분 방치에서 드랍→장착→수치 상승 확인.
- **3C 제단·유물 6빌드**: 피 수급(1/5/100×계수), 게이지·소환(요구량 점증),
  미보유 가중 2배, 5회마다 지정 소환, ★승급(3★+는 보스 처치 조건), 유물 스타일 머신 —
  6빌드(망령 군주/피의 광전사/역병 의사/순교자/처형자/왕의 그림자)의 공격 스타일·
  기본 규칙·3★·5★ 효과. 규칙은 데미지 파이프라인의 명시된 단계에만 후킹.
  완료: 6빌드 각각 장착 시 전투 로그·수치로 구분 동작, 테스트 통과.
- **3D 표준 더미 시뮬(core/sim.ts)**: 고정 시드 60초 빌드 스냅샷 시뮬 → 총피해 점수.
  장비 비교 증감과 기록(CP)에 연결. 완료: 동일 스냅샷 = 항상 동일 점수(결정성 테스트).
- **3E 스테이지·보스**: 60스테이지(드랍 풀 편향 = 사냥터), 도전/풀오토 연속 도전,
  보스 반복전(피 대량 + 해당 SIN 장비 확률↑), 보스 6종 기믹(바이블 명세), 도전 실패
  판정 2종 리포트, 환생 회차 보스 modifier. 레오니드 제단 카운터(전조 5초·피 소모 약화).
  완료: 1챕터 전 구간 + 루시안전이 기믹대로 동작.
- **3F 방치 보상**: 분당 수익 근사(골드·EXP·장비 요약·피 50%, 피 캡 = 소환 1회분),
  캡 10h, 복귀 데이터 생성. 완료: 탭 닫았다 열면 정산 데이터가 정확.

### Phase 4 — UI (UI_제작명세.md + 목업 v5 재현)
- **4A 콘솔 셸**: ConsoleShell/Bezel/LcdScreen/Deck/BloodLed/DmgSwitch — 목업 CSS 포팅,
  PixiJS 캔버스 마운트, 토글 패널 골격. 완료: 목업과 픽셀 단위 비교로 동일.
- **4B 패널 3종**: STAT/GEAR/ALTAR — Win·MenuItem·GbBar·Cell·RelicCard 컴포넌트,
  core 상태 바인딩(명세 8절 표) + **외형 선택 화면**(명세 9절). 완료: 실데이터로 목업과 동일 렌더, 스크롤 0.
- **4C 팝업·연출**: 비교/실패/복귀 팝업, 복귀 시퀀스(카운트업→피→"제단이 깨어났다"),
  전설·유물 연출(플래시+정지+팡파레 훅), 보이스 라인 시스템(담담체 1줄 표시).
  완료: 페이싱 문서의 "복귀 30초 내 소환+구매" 흐름이 실제로 이어짐.

### Phase 5 — 코드 아트 (PhaseV 문서 V-A~V-D 그대로)
저해상도 렌더+팔레트 → ASCII 스프라이트(생존자 5종=survivors_sprites.ts 기준 모션 확장,
몬스터 18종=monsters_sprites.ts **무외곽선 문법 변환 포함**, 보스 6종 24px+ 신규) →
절차 지형·입자(아트보드 챕터별 구도 목표) → 8비트 연출(히트스톱·점멸·체커보드 와이프) +
절망 빌드 팔레트 반전. 완료: 1챕터가 아트보드 무드로 플레이된다.

### Phase 6 — 코드 사운드
Web Audio 효과음 8종 + BGM 시퀀서(단조 5음계, 챕터별 키), 4채널 제한, 음소거.
완료: 1분 관전에서 타격·처치·드랍·소환이 귀로 구분.

### Phase 7 — 통합·밸런스 패스
페이싱 북극성 표 검증(첫 유물 5~8분, 루시안 40~60분, 소환 주기 15~30분...),
balance.ts 튜닝, 저장 마이그레이션 점검, 성능 프로파일.
완료: 수직 슬라이스(1챕터 + 6빌드 중 2종 + 환생 1회)가 페이싱 목표 안에서 재미있다.

### v1.1 로드맵 (지금 구현 금지)
온라인 랭킹 3종 제출·조회(빌드 스냅샷 서버 재시뮬 검증), Capacitor 패키징.
