# 1페이지 — 프로젝트 구조와 규칙

## 한 줄 개요
모바일 우선 웹 기반 2D 횡스크롤 방치형 파밍 RPG. 1인 개발, AI 코딩 에이전트로 구현.

## 기술 스택 (변경 금지)
Vite + TypeScript + React(UI) + PixiJS v8(게임 렌더) + Zustand + IndexedDB/idb + Capacitor(추후) + vitest. **외부 이미지·오디오 파일 금지** — 비주얼은 코드 생성(ASCII 스프라이트·절차 지형·팔레트 스왑), 사운드는 Web Audio 합성.

## 폴더 책임
- `core/` — 순수 TS 로직(전투·스탯·레벨·환생·드랍·제단·인벤·스테이지·방치 보상·표준 더미 시뮬). **Pixi/React/DOM import 금지. 무작위성은 rng 주입.** 이 순수성은 v1.1 서버 검증 랭킹의 전제이므로 절대 규칙.
- `render/` — Pixi 렌더(스프라이트·지형·입자·연출). core 상태 구독해 그리기만.
- `ui/` — React(탭·패널·팝업). core 호출과 스토어 구독만.
- `data/` — 정의 데이터 전부(몬스터·스테이지·장비·옵션 풀·유물·팔레트·스프라이트·곡). 상수만.
- `audio/` — 효과음 합성(sfx.ts)·BGM 시퀀서(bgm.ts).
- `data/balance.ts` — 모든 튜닝 수치의 유일한 출처(경험치 곡선·환생 배율·드랍률·비용 곡선 등). 타 파일 매직 넘버 금지.

## 아키텍처 원칙
1. 고정 타임스텝 60tps 시뮬레이션 + 렌더 보간.
2. 모든 수치는 core/stats.ts 집계 파이프라인 경유(직접 가감 금지).
3. 저장(IndexedDB, 5초 디바운스, 새로고침 복원 필수): 레벨·경험치·스탯 분배, 외형 선택, 인벤·장착, 유물 보유·★·장착, 진행도·사냥터, 골드·피, 환생 기록, 랭킹용 기록 3종, 설정, 마지막 저장 시각.
4. core/ 공식·드랍·집계·시뮬은 vitest 단위 테스트 필수(rng 주입 결정적 테스트).
5. 몬스터 30 + 데미지 텍스트 50 동시 60fps. 텍스트·이펙트 오브젝트 풀.
6. 랭킹 대비 데이터 규율: 기록 3종(최고 레벨 / 표준 더미 점수 / 환생 시 도달 스테이지)은 갱신 시각과 함께 단조 증가로만 기록.

## 진행 상태
- 완료: Phase 1(골격·이동), Phase 2(자동 전투).
- 구현 대기(본 문서가 명세): 레벨·스탯 분배, 장비·골드 시스템, 피의 제단·유물 6빌드, 스테이지·보스, 환생, 방치 보상, 코드 아트(Phase V 문서 + 아트컨셉보드.html 팔레트 확정), 코드 사운드, 표준 더미 측정.
- v1.1 로드맵(지금 구현 금지, 데이터만 준비): 온라인 랭킹 3종 제출·조회.
- 컷 확정(구현 금지): 직업 3종, 골드 능력치 구매, 스킬 탭, 동료, 카드 수집, 파티, 진영 상성, 임시 계약 버프.

## 작업 규칙

# 2페이지 — 게임 정체성

## 한 줄 정의
"8비트 게임기에서 디아블로2가 나왔다면" — 작은 도트 생존자가 세상을 끝낸 왕을 죽이러 가는 횡스크롤 자동전투 파밍 게임.

## 메인 테마
"빛은 이미 패배했다. 세상을 끝낸 왕을 죽일 수 있는 것은, 인간을 버린 자들뿐이다." 플레이어는 영웅이 아니라 끝까지 살아남아 버린 자이며, 죄의 유물을 몸에 박아 넣고, 죽어도 제단이 육신을 갈아엎어 다시 토해낸다(환생) — 조금 더 강하고, 조금 더 뒤틀린 채로.

## 핵심 루프와 4가지 의사결정
자동 사냥(경험치·골드·장비·피) → 레벨업 스탯 분배 → 피가 차면 제단에서 유물 → 골드로 옵션 재분배·상점 → 스테이지 도전 → 벽(레벨 벽·보스 벽) → 파밍·빌드로 돌파하거나 **환생으로 도약** → 반복.
의사결정 넷: **어디서 캘까(사냥터) / 무엇을 입을까(장비·리롤) / 어떤 죄를 받아들일까(유물) / 언제 다시 태어날까(환생).** 전투 조작은 없다 — 최종 보스의 제단 카운터 단 한 곳을 제외하고.

## 이중 벽 리듬 (페이싱의 척추)
경험치 곡선에 의도적 무릎: 1~40 완만 → 41+ 급등(1차 벽) → 81+ 재급등(2차 벽). **레벨 벽과 챕터 보스 벽을 같은 지점에 정렬**해 "환생할 때"라는 신호가 하나로 들리게 한다. 환생 보너스(도달 기록 비례 경험치 배율)로 직전 벽까지 질주 → 새 벽 → 파밍·빌드로 전투력을 올려 더 높이 찍고 환생 → 더 큰 배율. "환생을 잘하려면 파밍을 잘해야 한다"로 두 루프가 맞물린다.

## 갭 디자인 (모든 표현 판단의 기준)
귀여움과 어둠을 층으로 분리, 한 요소 안에서 섞지 않는다.
- **귀여움의 층 = 생물.** 무외곽선 플랫 마이크로 치비: 밝은 면을 어두운 배경에 직접(외곽선 금지), 머리가 전체의 60%, 눈은 피부 위 어두운 1px 두 개(반드시 분리), 캐릭터당 3~4색, 걷기 1px 바운스, 죽음은 작게 뿅. 생존자 약 10×10. 몬스터는 챕터가 깊을수록 대형화(8px→20px), 6장 권속만 붉은 눈. 보스 6종은 최대형(24px+).
- **어둠의 층 = 세계.** 챕터당 단일 색상 5단 명암 램프(아트컨셉보드.html hex 확정 → data/palettes.ts), 텅 빈 공간·부유 입자·디더 어둠, 느린 단조 BGM, 담담체 1~2줄 텍스트("루시안은 더 이상 웃지 않는다.").

## 아트·사운드 헌법
내부 해상도 320×180, 정수 스케일, nearest, 좌표 정수 반올림. 그라데이션·블러·글로우·알파 페이드 금지(어둠은 디더, 전환은 체커보드 와이프). 등급색(파/노/보/주)만 UI·드랍 연출 예외. 타격 = 히트스톱 80ms + 백색 점멸 2프레임 + 2px 넉백. 전설·유물 = 화면 플래시 + 정지 + 팡파레(장착 유물과 같은 죄의 옵션이면 한 음 추가). 절망 빌드 폭주 = 생존자 팔레트 반전. 효과음 Web Audio 4채널, BGM 2채널 시퀀서(단조 5음계, BPM 70~85, 밀도 40%↓, 챕터별 키 교체).

## 세계관 구조 (6보스 서사)
꼭두각시 왕 레오니드는 딸을 잃은 밤 악마와 계약해 세상을 끝냈다. 6챕터 × 10스테이지 = 60:
① 핏빛 연회장(암적) 루시안 벨모어 ② 시체의 들판(회갈) 그라브몰 ③ 역병 도시(독녹) 마르세라 ④ 부러진 성당(회청) 카르디온 ⑤ 심연의 균열(암보라) 아자르 ⑥ 검은 왕좌(칠흑+적) 레오니드. 스토리는 챕터 진입·격파 시 담담체 텍스트만. 엔딩: 왕은 작게 누군가의 이름을 부른다. 환생의 서사 = "심연과 계약한 자는 한 번 죽어도 사라지지 않는다."

# 3페이지 — 시스템 명세

## 캐릭터·레벨·스탯
단일 생존자 — **외형 스킨 5종 중 선택**(방랑자/이끼머리 검사/붕대 이단자/깃털 투구병/수의의 사제, data/sprites/survivors.ts). 능력치·성장 완전 동일, 순수 외형. 첫 실행 시 선택 화면(GB 메뉴), 이후 STAT 패널에서 자유 변경. 처치 경험치(스테이지 계수) → 레벨업마다 **스탯 포인트 +3**, atk/def/maxHp/hpRegen에 분배. 자동 분배 프리셋(공격형/균형형/생존형) 기본, 수동 분배는 심화. 분배 리셋은 환생 시 무료. 경험치 곡선: 구간별 증가율 [1~40 완만 / 41~80 급등 / 81+ 재급등] — 무릎 위치·기울기는 balance.ts.

## 환생 (v1.0 코어)
- 해금: 루시안(첫 보스) 처치. 초기화: 레벨·스탯·스테이지 진행. **보존: 장비·유물·★·골드·도감·기록.**
- 보너스: 환생 시점 도달 레벨·전투력 비례 **영구 경험치 배율** + 환생 횟수당 영구 스탯 보너스 (곡선 balance.ts).
- 환생 직후 재돌파를 위한 **풀오토 도전(자동 연속 진행)** 필수 — 1챕터 원킬 질주가 이 시스템 최대의 파워 판타지.
- 회차별 보스 강화 modifier(원안 계승: 1회차 루시안 망령 3체 / 그라브몰 회복 1.3배 ...).
- 환생 기록 배열(회차, 도달 스테이지, 도달 레벨, 시각) 저장 — 환생 스테이지 랭킹의 원천.

## 성장 3축 + 골드
**① 레벨·스탯 분배** — 위. **② 장비(방치 사냥 드랍)** — 슬롯 4(weapon/helmet/armor/accessory), 등급 5(회/파/노/보/주). 기본 스탯 = 슬롯 기본치 × itemLevel × 등급 배율. 옵션 0/1/2/3/4줄 + **전설은 '죄의 옵션' 1줄 확정**(영웅 30%). 옵션 풀: critChance(상한75%)/critDamage/attackSpeed(상한+100%)/damageIncrease(합)/finalDamage(곱·희소)/defPenetration/lifeSteal(상한10%)/goldGain/damageReduction(상한50%) + 죄의 옵션 6계열(유물 표 참조). 슬롯 제한 유지. 드랍 8%/처치, 가중치 60/25/10/4/1(챕터 상향), 인벤 60칸·자동 정리·자동/일괄 판매(전설 제외). 큐브: 동일 등급 3→상위 1. **③ 죄의 유물(피의 제단)** — 아래.
**골드 사용처(능력치 구매 없음)**: ⓐ **옵션 재분배** — 장비의 일반 옵션 리롤(죄의 옵션은 보존), 동일 장비 연속 사용 시 비용 점증. ⓑ **랜덤 상점** — 매물 6칸, 시간 갱신 무료 + 골드 즉시 갱신, 낮은 확률로 죄의 옵션 확정 매물.

## 피의 제단과 죄의 유물 (6빌드)
피: 일반 1 / 정예 5 / 보스 100(스테이지 계수). 게이지 충전 → 소환 1회(요구량 점증). **미보유 가중 2배, 소환 5회마다 지정 소환(죄 계열 선택) 적립.** 중복 = ★+1(상한 5★), **3★ 이상 승급은 해당 죄의 보스 처치 후 개방.** 유물 슬롯 1 — 장착 유물이 공격 스타일·규칙·비주얼을 정의(미장착 = 약한 맨손 단타).

## 보스 6종 (기믹 원안, 수치 balance.ts)

## 데미지 공식 (순서 변경 금지, 테스트 고정)
raw = atk × styleMult → ×(1+damageIncrease합) → 유효방어 = max(0, def−관통), ×(1−유효방어/(유효방어+100)) → 치명 판정(×critDamage) → ×(1+finalDamage) → ×(1−상대 damageReduction) → ±5% 분산, 최소 1. 유물 규칙은 명시된 단계에만 후킹. 처형·비율 피해는 별도 채널(공식 우회를 명시적으로 선언).
